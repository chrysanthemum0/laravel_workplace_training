*마이그레이션
마이그레이션이란 sql을 쓰는 것이 아닌, 프로그램에서 데이터 베이스의 테이블을 관리하는 구조임
운용중 테이블의 정의의 수정이 필요할때 유요함 
라라벨뿐만 아니라 웹 프레임워크에서는 기본적으로 테이블조작은 마이그레이션으로, 데이터조작은 O/R맵핑
으로 각각 관리함 

*마이그레이션의 상황 확인 
php artisan migrate:status -> 현재 마이그레이션 스태터스 확인 가능
status는 batch/status 두 종류가 있음 
status : ran->실행중, pending->마이그레이션해야됨 
batch : 몇회 마이그레이션 되었는지 

*php artisan migrate:rollback
- 마이그레이션을 다시 하는 기능 

*마이그레이션의 up(), down()메소드
업메소드는 마이그레이션을 실행 시 / 다운은 롤백 실행 시 실행됨 
그렇기에 두가지의 처리는 반드시 필요함 

*migrate:reset
-php artisan migrate:reset -> 모든 마이그레이션의 다운 메소드를 실행 
-php artisan migrate:refresh -> reset후 마이그레이션 
-php artisan migrate:fresh -> 테이블을 모두 지웠다가 다시 한번 마이그레이션
참고 157p

*롤백은 신중하게
롤백은 테이블 관리의 이력을 없던일로 하는 것임 
또한, 실행 내용에 따라 보존 되어있는 데이터를 잃을 수 도 있음
팀 단위 개발을 진행할 때도 잘못된 실행을 하면 막대한 손해와 민폐를 끼칠 수 있으므로 반드시 주의 

*컬럼 추가 마이그레이션
-php artisan make:migration 파일명 --table=테이블명

*컬럼 변경 마이그레이션
라이브러리 설치 후 진행 참고 161p 

*Seeder
-DB에 다시 한번 새로운 초기 데이터를 등록하는 것 참고 166페이지 

*Eloquent(O/R 맵핑)
ORM이란 기능이 라라벨에 있음 
ORM이란 Object-Relational-Mapping의 줄임말로 프로그래밍언어와 데이터베이스 간의
비상관 데이터 조작을 하는 방법임 
모델의 객체와 데이터베이스의 테이블을 매핑시켜주기 위해 자동으로 SQL문을 생성해준다.

*중요하니깐 170p부터 다시 정리 할 것
